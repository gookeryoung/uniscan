## 关键决策与依据

1. **GitHub Desktop 风格而非杀毒软件风格**：原 main_window.py 采用「模式卡片 + 大扫描按钮」杀软风格，用户明确要求改为 GitHub Desktop 风格（菜单→主操作区→左右分栏列表+详情）。本规则作为后续 GUI 重构依据，本轮不动 GUI 代码。
2. **5 区顶层划分**：① 菜单栏、② 主操作区、③ 列表区、④ 详情区、⑤ 底部操作区。其中 ⑤ 嵌套在 ③ 内部底部，④ 包含操作栏与主体两子区。编号沿用用户需求顺序。
3. **列表区强制 Tab 切换**：用 `QTabWidget` 支持多视图（扫描结果/规则文件/扫描历史），每 Tab 独立选中态持久化，避免单列表切换丢失上下文。
4. **底部操作区输入框+按钮组**：对应 GitHub Desktop 左下角 commit message 输入框 + 提交按钮形态，明确要求 `QPlainTextEdit` + `QHBoxLayout` 按钮组，按钮与列表选中联动。
5. **详情操作栏强制 QStackedWidget**：空态/非空态两页面持久化，禁止动态 `add/remove` 控件，避免重建导致的闪烁与状态丢失。用户明确选择「强制 QStackedWidget」。
6. **ASCII 示意图用 Unicode 框线**：用户要求「包含示意图」，用 `┌─┐│└┘├┤` 字符绘制 5 区位置关系，下方配 5 区职责列表辅助理解。
7. **配色与字体层级独立子节**：GitHub Desktop 配色（#f6f8fa 背景 / #0366d6 主色 / #d73a49 危险色）与字体层级（14/13/13/12px）单独成节，与既有「样式（QSS）」章节交叉引用，不重复 QSS 语法细节。
8. **不动 GUI 代码**：本轮仅完善规则文件，GUI 重构作为后续迭代单独进行，避免规则与实现同时变更导致难以验证。
