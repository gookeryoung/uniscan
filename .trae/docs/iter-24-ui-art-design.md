# iter-24: UI 美术设计优化实施计划

## 迭代目标

针对 fuscan 项目的 UI 进行系统性美术设计优化，解决以下问题：

1. **窗口初始位置始终不好看**：首次启动在 (0,0)，无居中逻辑；恢复位置时未 clamp 到屏幕边界
2. **界面宽度过宽**：960px 初始宽度对扫描工具而言过于宽松，内容密度低
3. **操作区域不和谐**：顶部两行 QGroupBox 边框 + 箭头标签导致视觉碎片化

## 设计系统规格

### 间距系统

| 用途 | 尺寸 |
|------|------|
| 控件内边距 | 4px |
| 行内控件间距 | 6px |
| 区块间距 | 8px |
| 卡片边距 | 8px |

### 颜色系统

| 用途 | 颜色值 |
|------|--------|
| 背景 | #f6f8fa |
| 卡片背景 | #ffffff |
| 边框 | #e1e4e8 |
| 主色 | #0366d6 |
| 成功色 | #2ea44f |
| 文本 | #24292e |
| 次要文本 | #586069 |
| 禁用文本 | #959da5 |

### 控件高度

- 统一所有按钮/输入控件高度为 32px（除扫描按钮保持 36px 突出）

## 实施步骤

### Step 1: 窗口几何优化

- 首次启动（无保存配置）：居中于主屏幕
- 后续启动：恢复上次位置，但 x/y 需 clamp 到可用屏幕边界内
- 初始尺寸：960×640 → 800×560
- 最小尺寸：800×500 → 720×480

### Step 2: 顶部操作区重构

- 移除 QGroupBox 的边框和标题，改用扁平化卡片容器
- 将两个 QGroupBox 合并为一个统一的 `control_card` 容器
- 移除箭头标签，改用统一间距（8px）替代
- 三个功能区：扫描模式+目标 / 规则 / 扫描按钮，用间距自然分隔

### Step 3: QSS 统一

- 统一所有控件高度为 32px
- 统一边框颜色为 #e1e4e8
- 统一圆角为 6px（卡片）/ 4px（控件）

## 改动文件清单

- `main_window.ui`：修改窗口尺寸、移除箭头标签、合并容器结构
- `main_window_ui.py`：重新编译
- `main_window.py`：添加窗口居中逻辑、移除箭头标签绑定、调整布局参数
- `styles.qss`：统一规格、移除箭头样式、添加 control_card 样式
- `tests/test_gui.py`：更新相关测试

- 窗口在 720×480 最小尺寸下无内容溢出
