# req-20：扫描体验优化

## 需求清单

[x] 需求1：损坏压缩包日志精简（去掉 exc_info=True，仅记录简洁 warning）
[x] 需求2：暂停/取消按钮改为常规按钮背景，不要红色
[x] 需求3：取消响应优化——点击后立即显示"取消中"（转圈），后台异步调用取消操作
[x] 需求4：扫描过程分为四阶段（准备扫描、解析目录、文件解析、扫描完成），下方提示显式体现，不同阶段切换与侧边栏功能禁用状态关联
[x] 需求5：解析目录阶段按所选文件类型和忽略规则快速分析出需解析的文件（白名单制度），提高执行速度
[x] 需求6：压缩文件（zip/7z/rar 等）单独列分类，不在配置中设计；不勾选则解析目录后不计待解析数量，下阶段不解析
[x] 需求7：设置中的忽略项移到配置页文件类型右侧，通过 TAB 切换忽略目录和忽略扩展名

## 实施顺序

1. 需求1：日志精简（独立，最简单）
2. 需求2：按钮样式（独立，仅 QSS）
3. 需求6：压缩文件独立分类（需求4/5 的前置依赖）
4. 需求5：解析目录白名单优化（依赖需求6）
5. 需求4：扫描四阶段 + 侧边栏关联（依赖需求5/6）
6. 需求3：取消响应优化（独立）
7. 需求7：忽略项位置调整（独立 UI 重构）
