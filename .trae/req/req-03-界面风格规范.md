# 需求03：界面风格规范（GitHub Desktop 风格）

- [x] 1. 顶部包含菜单栏，菜单按业务域分组，菜单 action 与工具栏、快捷键复用同一 QAction 实例。
- [x] 2. 菜单下方按照功能业务流程分配主操作区，承载核心操作（扫描模式、目标、规则、扫描按钮、进度、统计等），高度固定、不随列表滚动。
- [x] 3. 主操作区下方为左右分栏：左侧为列表区，顶部用 QTabWidget 可切换不同 Tab 视图；右侧为详情区。
- [x] 4. 列表区底部为具体操作区域，包含提交信息类多行输入框与按钮组（导出/批量处理等），固定在列表区底部、不随列表滚动。
- [x] 5. 详情区上部提供操作区域：详情内容为空时提供全局操作（引导开始扫描、加载规则等），不为空时针对具体详情提供操作（定位命中、上下条跳转、打开文件位置等）；两态切换强制用 QStackedWidget。
